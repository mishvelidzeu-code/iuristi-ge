# პროექტის მიმოხილვა — იურისტი

ბოლო განახლება: 12 ივლისი, 2026

## მიზანი

„იურისტი“ არის ქართული სამართლებრივი ტესტების პლატფორმა. მომხმარებელს შეუძლია სწავლა სამართლის მიმართულების, კონკრეტული კანონის, თემისა და სირთულის მიხედვით. სისტემა მოიცავს ავტორიზაციას, შედეგებს, ქულებს, რეიტინგს, PRIME არქიტექტურას, ადმინისტრირებას და სამართლებრივი წყაროების კონტროლს.

## მონაცემთა ბაზა

ძირითადი ცხრილებია:

- მომხმარებლები: `profiles`, `subscriptions`, `user_achievements`;
- კატალოგი: `categories`, `subcategories`, `tests`, `test_questions`;
- კითხვები: `questions`, `question_options`, `saved_questions`, `reports`;
- კანონები: `laws`, `law_articles`, `question_law_articles`;
- მცდელობები: `test_attempts`, `user_answers`, `daily_usage`;
- ოპერაციები: `payments`, `certificates`, `audit_logs`, `system_settings`.

ყველა პირადი და ადმინისტრაციული მონაცემი დაცულია RLS პოლიტიკებით. ქულების დათვლა, დღიური ლიმიტი და პასუხების შეფასება გათვალისწინებულია უსაფრთხო PostgreSQL RPC ფუნქციებში.

## კანონების მიხედვით ტესტები

სისტემაში საწყისად დამატებულია:

1. საქართველოს კონსტიტუცია;
2. საქართველოს ზოგადი ადმინისტრაციული კოდექსი;
3. საქართველოს ადმინისტრაციულ სამართალდარღვევათა კოდექსი;
4. საქართველოს სისხლის სამართლის კოდექსი;
5. საქართველოს სისხლის სამართლის საპროცესო კოდექსი;
6. საქართველოს სამოქალაქო კოდექსი;
7. საქართველოს სამოქალაქო საპროცესო კოდექსი.

თითოეული კითხვა შეიძლება დაუკავშირდეს ერთ ძირითად კანონსა და რამდენიმე მუხლს. Production კითხვა ქვეყნდება მხოლოდ გადამოწმების, წყაროსა და რედაქციის თარიღის მითითების შემდეგ.

## ავტორიზაცია და ადმინისტრირება

- ავტორიზაცია მუშაობს Supabase Auth-ზე;
- ადმინისტრატორის უფლებები მოწმდება frontend-სა და database-ის დონეზე;
- პირველი ადმინისტრატორის ანგარიში უკვე შექმნილია;
- `service_role` გასაღები frontend-ში არ ინახება;
- საიტი იყენებს მხოლოდ საჯარო Supabase publishable key-ს.

## მიმდინარე მდგომარეობა

- Supabase-ის ძირითადი და კანონების მიგრაციები გაშვებულია;
- მონაცემთა ბაზის კავშირი აქტიურია;
- GitHub Pages deployment აქტიურია;
- desktop/mobile header გამოსწორებულია;
- კანონების კატალოგის API აბრუნებს 7 კანონს;
- საქართველოს კონსტიტუციის სრული 143 კითხვა მომზადებულია CSV ფორმატში 78-ვე მუხლის დაფარვით;
- HTML და JavaScript შემოწმებები წარმატებით გადის.

## შემდეგი პრიორიტეტები

1. ადმინისტრატორისთვის კანონებისა და მუხლების CRUD ინტერფეისი;
2. ზოგადი ადმინისტრაციული კოდექსის გადამოწმებული კითხვების პირველი პაკეტი;
3. Supabase-ში სრული 143-კითხვიანი კონსტიტუციის CSV-ის იმპორტი, როცა CLI/DB წვდომა ხელახლა იქნება მინიჭებული;
4. კანონის თავებისა და მუხლების მიხედვით ფილტრები;
5. შედეგების Supabase-ში შენახვა და პროფილში რეალური სტატისტიკა;
6. PRIME გადახდის backend და ვერიფიცირებული webhook;
7. სერტიფიკატის server-side გენერაცია;
8. production დომენი, CSP, monitoring და ავტომატური backup.
